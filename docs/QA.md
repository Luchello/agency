# ✅ QA (Quality Assurance) 프레임워크

## 철학
> 납품 전에 우리가 고객보다 더 까다롭게 검수한다.
> 문제는 우리 안에서 발견하고 고친다. 고객에게 문제를 넘기지 않는다.

## QA 루프 (자가개선 루프 기반)
```
1차 감사 → 수정 → 2차 감사 → 수정 → 최종 확인 → 납품 승인
```
최소 2회전. 문제 발견 시 3회전까지.

---

## 1차 감사: 자동 검사

### 🔧 빌드 검사
```bash
npm run build 2>&1
# 에러 0, 워닝 확인
```
- [ ] 빌드 성공 (에러 없음)
- [ ] 빌드 워닝 0개 (또는 사유 확인됨)
- [ ] dist 사이즈 적정 (JS < 300KB gzip)

### 🔗 링크 검사
```bash
# 모든 href 확인
grep -rn 'href=' src/ | grep -v node_modules
```
- [ ] 깨진 링크 없음
- [ ] 외부 링크 target="_blank" + rel="noopener"
- [ ] 전화번호 링크 정확 (tel:)
- [ ] 이메일 링크 정확 (mailto:)

### 📱 반응형 검사
- [ ] 모바일 (320px~480px)
- [ ] 태블릿 (768px)
- [ ] 데스크톱 (1024px+)
- [ ] 가로 스크롤 없음
- [ ] 텍스트 잘림 없음
- [ ] 터치 타겟 44px 이상

### ♿ 접근성 검사
- [ ] heading 계층 (h1→h2→h3, 건너뛰기 없음)
- [ ] 모든 img에 alt 텍스트
- [ ] 인터랙티브 요소에 aria-label
- [ ] focus-visible 스타일 적용
- [ ] 키보드로 전체 네비게이션 가능
- [ ] 색상 대비 WCAG AA 이상
- [ ] skip-to-content 링크

### 🔍 SEO 검사
- [ ] title 태그 (50-60자, 키워드 포함)
- [ ] meta description (120-160자)
- [ ] OG 태그 (title, description, image, url)
- [ ] canonical URL
- [ ] robots.txt
- [ ] sitemap.xml
- [ ] JSON-LD 구조화 데이터 (해당 시)
- [ ] heading에 키워드 자연스럽게 포함

### ⚡ 성능 검사
- [ ] 이미지 최적화 (WebP, 적정 크기)
- [ ] lazy loading 적용 (below-fold 이미지)
- [ ] 코드 스플리팅 (React.lazy)
- [ ] hero 이미지 preload
- [ ] 불필요한 패키지 없음
- [ ] CSS 미사용 클래스 없음

### 🛡️ 보안 검사
- [ ] 민감한 정보 하드코딩 없음 (API 키 등)
- [ ] form에 CSRF 보호 (해당 시)
- [ ] HTTPS 적용
- [ ] 의존성 취약점 확인 (`npm audit`)

## 2차 감사: 콘텐츠 & UX 수동 검수

### 📝 콘텐츠 검사
- [ ] 오타 없음
- [ ] 업체명/전화번호/주소 정확
- [ ] 서비스 설명 정확 (고객 확인 사항)
- [ ] 이미지 적절 (깨지지 않음, 해상도 OK)
- [ ] 저작권 문제 없음 (이미지 출처)

### 🎨 디자인 검사
- [ ] 색상 일관성
- [ ] 폰트 일관성
- [ ] 간격/여백 일관성
- [ ] 호버/애니메이션 자연스러움
- [ ] 로딩 상태 존재 (스피너/쉬머)
- [ ] 에러 상태 처리 (404, 에러 바운더리)

### 🖱️ 기능 검사
- [ ] 네비게이션 모든 링크 작동
- [ ] 문의 폼 제출 → 결과 확인
- [ ] 전화 버튼 → 실제 전화 연결
- [ ] 모바일 메뉴 열기/닫기
- [ ] 스크롤 애니메이션 작동
- [ ] 라이트박스/모달 열기/닫기/ESC

### 🌐 크로스 브라우저
- [ ] Chrome (필수)
- [ ] Safari (필수 — iOS 사용자 많음)
- [ ] Firefox (권장)
- [ ] Samsung Internet (권장 — 한국 안드로이드)
